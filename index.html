BsProse — live demo

A tiny WYSIWYG editor for Bootstrap 5 sites: a single file, no build step, no dependencies. It lets you write rich prose and drop in native Bootstrap components (alert, card, accordion, grid, carousel...) that survive editing byte-for-byte.

Introduction

This page hosts several independent editors, each configured differently to showcase the library's key features. Under every editor there is a Result block — the live output of getHTML(), updated through the onChange callback.

ⓘ
BsProse injects ~30 lines of CSS (id bsprose-css) built on Bootstrap CSS variables, so no separate stylesheet is needed. The Bootstrap JS bundle is optional — it only powers the live preview of interactive components (accordion / carousel / tabs) inside the editor.

1. Full editor out of the box

Default toolbar: format, B/I/U, colors, alignment, links, lists, image-by-URL, video, components, grid and the HTML toggle.

Nothing extra is passed here — just value, placeholder, the BsProse.imageUrl() image source and onChange. Every other option (grid, embed, sourceToggle) defaults to on. 

// Example 1 — default configuration
BsProse.create('#editor-1', {
  value: '<h2>Hello!</h2>This is <b>BsProse</b>.',
  placeholder: 'Start writing your page…',
  image: { sources: [ BsProse.imageUrl() ] },
  onChange: html => out1.textContent = html
});

3. Multiple image sources

            

              URL, file upload (imageUpload) and a custom "gallery" source.
            

          

          

            

              With several sources, the "image" button shows a chooser first. Upload points to
              the bundled upload.php; the custom source is just an object
              { id, label, open(api) } that calls api.insert(spec).
            

            
// Example 3 — three image sources
const gallery = {
  id: 'gallery', label: 'Media library (demo)',

    });
  }
};
BsProse.create('#editor-3', {

4. Custom component catalog

            

              Extend BsProse.components with a custom "Pricing card" template.
            

          

          

            

              A component is { id, group, label, html }. Any {{id}}
              token in the template is replaced with a unique id on insert. Here a marketing
              block is added to the default set.
            

            
// Example 4 — extending the component palette
BsProse.create('#editor-4', {
  components: [
    ...BsProse.components,                // keep the defaults
    {
      id: 'pricing', group: 'Marketing', label: 'Pricing card',

            '<a class="btn btn-primary">Choose</a></div></div>'

6. Responsive images: srcset & <picture>

            

              The image spec can carry srcset/sizes (a DPR-aware
              <img>) or sources (a viewport-driven <picture>).
            

          

          

            

              Any image source (URL, upload or a custom gallery) resolves an
              image spec. Beyond src you may add:
            

            
    
              
  • 
                srcset + sizes — the editor outputs a plain
                <img srcset sizes>. The browser picks a candidate by the
                rendered size × device-pixel-ratio — great for HiDPI/Retina,
                but it never downsizes once a larger file is cached.
              
    • 
              
  • 
                sources — an array of
                { media, type, srcset, sizes }. The editor emits a
                <picture> with <source media> rules: a
                hard, viewport-driven switch re-evaluated on resize (narrower
                window → smaller file). The plain <img> becomes the
                largest/default fallback and drops its own srcset/sizes.
              
    • 
            

            

              The two custom sources below insert each variant. Open the image chooser and pick
              one, then inspect the getHTML() output to see <img srcset>
              versus <picture>.
            

            
// Example 6 — two responsive-image specs

// (a) DPR-aware <img srcset sizes>
const srcsetSource = {
  id: 'srcset', label: 'Insert <img srcset>',
  open(api) {
    api.insert({
      src:    'https://placehold.co/1280x720',
      srcset: 'https://placehold.co/480x270 480w, ' +
              'https://placehold.co/960x540 960w, ' +
              'https://placehold.co/1280x720 1280w',
      sizes:  '(max-width: 960px) 100vw, 960px',
      width: 1280, height: 720, alt: 'Responsive photo',
      class: 'img-fluid rounded shadow-sm'
    });
  }
};

// (b) viewport-driven <picture> via `sources`
const pictureSource = {
  id: 'picture', label: 'Insert <picture>',
  open(api) {
    api.insert({
      src: 'https://placehold.co/1280x720',   // fallback / largest
      sources: [
        { media: '(max-width: 575.98px)',  srcset: 'https://placehold.co/480x270' },
        { media: '(max-width: 1199.98px)', srcset: 'https://placehold.co/960x540' }
      ],
      width: 1280, height: 720, alt: 'Art-directed photo',
      class: 'img-fluid rounded shadow-sm'
    });
  }
};

BsProse.create('#editor-6', {
  image: { sources: [ srcsetSource, pictureSource ] },
  toolbar: ['format', '|', 'bold', 'italic', '|', 'align', '|', 'image'],
  onChange: html => out6.textContent = html
});

            
            

            

              getHTML() result
              

            

            

              Rule of thumb: pass only srcset for the DPR-aware default; pass
              sources when you want a hard, art-directed switch by viewport.
