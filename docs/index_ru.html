bsprose.js · v1.2.5

BsProse — живое демо

Крошечный WYSIWYG-редактор для сайтов на Bootstrap 5: один файл, без сборки, без зависимостей. Он позволяет писать форматированный текст и вставлять нативные компоненты Bootstrap (alert, card, accordion, grid, carousel...), которые переживают редактирование байт-в-байт.

  value: html,

Введение

На этой странице размещено несколько независимых редакторов, каждый из которых настроен по-своему, чтобы показать ключевые возможности библиотеки. Под каждым редактором есть блок Результат — живой вывод getHTML(), обновляемый через колбэк onChange.

ⓘ
BsProse внедряет ~30 строк CSS (id bsprose-css), построенных на CSS-переменных Bootstrap, поэтому отдельный стиль не нужен. JS-бандл Bootstrap необязателен — он лишь обеспечивает живой предпросмотр интерактивных компонентов (accordion / carousel / tabs) внутри редактора.

1. Полный редактор «из коробки»

Панель по умолчанию: формат, B/I/U, цвета, выравнивание, ссылки, списки, изображение по URL, видео, компоненты, сетка и переключение HTML.

Здесь не передаётся ничего лишнего — только value, placeholder, источник изображений BsProse.imageUrl() и onChange. Все остальные опции (grid, embed, sourceToggle) включены по умолчанию. 

// Пример 1 — конфигурация по умолчанию
BsProse.create('#editor-1', {
  value: '<h2>Hello!</h2>This is <b>BsProse</b>.',
  placeholder: 'Начните писать свою страницу…',
  image: { sources: [ BsProse.imageUrl() ] },
  onChange: html => out1.textContent = html
});

2. Минимальная панель (комментарии блога)

            

              Урезанная toolbar, размер sm, без компонентов, сетки,
              видео и переключения HTML.
            

          

          

            

              Мы передаём свой массив токенов toolbar и отключаем тяжёлые
              возможности. Кнопки, чья функция не настроена, пропускаются автоматически.
            

            
// Пример 2 — компактный набор инструментов
BsProse.create('#editor-2', {
  toolbar: ['bold', 'italic', 'underline', '|',
            'link', 'bullet', 'number', '|', 'clean'],
  toolbarSize: 'sm',
  grid: false, embed: false, sourceToggle: false,
  placeholder: 'Ваш комментарий…',
  onChange: html => out2.textContent = html
});

3. Несколько источников изображений

            

              URL, загрузка файла (imageUpload) и свой источник «галерея».
            

          

          

            

              При нескольких источниках кнопка «изображение» сначала показывает выбор. Загрузка
              указывает на встроенный upload.php; свой источник — это просто объект
              { id, label, open(api) }, который вызывает api.insert(spec).
            

            
// Пример 3 — три источника изображений
const gallery = {
  id: 'gallery', label: 'Медиатека (демо)',
  open(api) {
    api.insert({
      src: 'https://placehold.co/640x360',
      alt: 'Случайное фото', title: 'Случайное фото', class: 'img-fluid rounded shadow-sm'
    });
  }
};
BsProse.create('#editor-3', {
  image: { sources: [
    BsProse.imageUrl(),
    BsProse.imageUpload({ endpoint: 'upload.php' }),
    gallery
  ]},
  toolbar: ['format', '|', 'bold', 'italic', '|', 'align', 'link', '|', 'image'],
  onChange: html => out3.textContent = html
});

4. Каталог своих компонентов

            

              Расширьте BsProse.components своим шаблоном «Карточка тарифа».
            

          

          

            

              Компонент — это { id, group, label, html }. Любой токен {{id}}
              в шаблоне заменяется уникальным id при вставке. Здесь маркетинговый
              блок добавлен к набору по умолчанию.
            

            
// Пример 4 — расширение палитры компонентов
BsProse.create('#editor-4', {
  components: [
    ...BsProse.components,                // сохраняем стандартные
    {
      id: 'pricing', group: 'Маркетинг', label: 'Карточка тарифа',
      html: '<div class="card text-center shadow-sm">' +
            '<div class="card-header">Pro</div>' +
            '<div class="card-body"><h3>$29</h3>' +
            '<a class="btn btn-primary">Choose</a></div></div>'
    }
  ],
  onChange: html => out4.textContent = html
});

6. Адаптивные изображения: srcset & <picture>

            

              Спецификация изображения может нести srcset/sizes (DPR-зависимый
              <img>) или sources (зависимый от вьюпорта <picture>).
            

          

          

            

              Любой источник изображения (URL, загрузка или своя галерея) разрешается в
              спецификацию изображения. Помимо src можно добавить:
            

            
    
              
  • 
                srcset + sizes — редактор выводит обычный
                <img srcset sizes>. Браузер выбирает кандидата по
                отображаемому размеру × device-pixel-ratio — отлично для HiDPI/Retina,
                но он никогда не уменьшает изображение, если больший файл уже закэширован.
              
    • 
              
  • 
                sources — массив
                { media, type, srcset, sizes }. Редактор выдаёт
                <picture> с правилами <source media>:
                жёсткое переключение по вьюпорту, переоцениваемое при изменении размера (уже
                окно → меньший файл). Обычный <img> становится
                наибольшим/запасным вариантом и отбрасывает свои srcset/sizes.
              
    • 
            

            

              Два своих источника ниже вставляют каждый вариант. Откройте выбор изображения и выберите
              один, затем изучите вывод getHTML(), чтобы увидеть <img srcset>
              против <picture>.
            

            
// Пример 6 — две спецификации адаптивных изображений

// (a) DPR-зависимый <img srcset sizes>
const srcsetSource = {
  id: 'srcset', label: 'Вставить <img srcset>',
  open(api) {
    api.insert({
      src:    'https://placehold.co/1280x720',
      srcset: 'https://placehold.co/480x270 480w, ' +
              'https://placehold.co/960x540 960w, ' +
              'https://placehold.co/1280x720 1280w',
      sizes:  '(max-width: 960px) 100vw, 960px',
      width: 1280, height: 720, alt: 'Адаптивное фото', title: 'Адаптивное фото',
      class: 'img-fluid rounded shadow-sm'
    });
  }
};

// (b) зависимый от вьюпорта <picture> через `sources`
const pictureSource = {
  id: 'picture', label: 'Вставить <picture>',
  open(api) {
    api.insert({
      src: 'https://placehold.co/1280x720',   // запасной / наибольший
      sources: [
        { media: '(max-width: 575.98px)',  srcset: 'https://placehold.co/480x270' },
        { media: '(max-width: 1199.98px)', srcset: 'https://placehold.co/960x540' }
      ],
      width: 1280, height: 720, alt: 'Художественно скадрированное фото', title: 'Художественно скадрированное фото',
      class: 'img-fluid rounded shadow-sm'
    });
  }
};

BsProse.create('#editor-6', {
  image: { sources: [ srcsetSource, pictureSource ] },
  toolbar: ['format', '|', 'bold', 'italic', '|', 'align', '|', 'image'],
  onChange: html => out6.textContent = html
});

            
            

            

              Результат getHTML()
              

            

            

              Правило: передавайте только srcset для DPR-зависимого варианта по умолчанию; передавайте
              sources, когда нужно жёсткое, художественное переключение по вьюпорту.

ОпцияТипПо умолчаниюЗначение

            
            
            
valuestring''Начальный HTML.

            
onChange(html)=>void—Вызывается при изменениях (дебаунс ~120мс).

            
placeholderstringStart writing…Показывается, когда редактор пуст.

            
toolbarstring[]полный наборПорядок токенов панели; '|' — разделитель.

image{ sources:[] }undefinedИсточники изображений; иначе кнопка скрыта.

            
gridbooleantrueВизуальный конструктор сетки.

            
embedbooleantrueВставка видео в блок ratio.

            
toolbarSizesm|md|lgmdРазмер элементов панели.

            
historyLimitnumber120Глубина undo/redo.

            
sourceTogglebooleantrueПереключение HTML-источника (всегда справа).

            
          

        

        

          Токены панели: format, bold, italic,
          underline, color, bgcolor, align,
          link, bullet, number, image,
          embed, component, grid, clean,
          |.
        

        

          Поля спецификации изображения: src (обязательно), srcset, sizes
          (→ адаптивный <img>), sources (→ <picture>),
          alt, title, width, height, class. См. пример 6.
