BsProse — สาธิตสด

โปรแกรมแก้ไข WYSIWYG ขนาดเล็กสำหรับเว็บไซต์ Bootstrap 5: ไฟล์เดียว ไม่ต้องบิลด์ ไม่มีการพึ่งพา ช่วยให้คุณเขียนข้อความที่มีรูปแบบ และ แทรก คอมโพเนนต์ Bootstrap ต้นฉบับ (alert, card, accordion, grid, carousel...) ที่ยังคงสมบูรณ์ ทุกไบต์ หลังการแก้ไข

บทนำ

หน้านี้มี โปรแกรมแก้ไขอิสระ หลายตัว แต่ละตัวตั้งค่าแตกต่างกัน เพื่อแสดงคุณสมบัติหลักของไลบรารี ใต้โปรแกรมแก้ไขแต่ละตัวมีบล็อก ผลลัพธ์ — ผลลัพธ์สดของ getHTML() อัปเดตผ่านคอลแบ็ก onChange

ⓘ
BsProse แทรก CSS ประมาณ 30 บรรทัด (id bsprose-css) ที่สร้างบน CSS variables ของ Bootstrap จึงไม่จำเป็นต้องใช้สไตล์ชีตแยกต่างหาก JS bundle ของ Bootstrap เป็นตัวเลือก — ใช้งานเฉพาะสำหรับพรีวิวสดของคอมโพเนนต์แบบอินเทอร์แอคทีฟ (accordion / carousel / tabs) ภายในโปรแกรมแก้ไข

1. โปรแกรมแก้ไขเต็มรูปแบบพร้อมใช้งาน

แถบเครื่องมือเริ่มต้น: รูปแบบ, B/I/U, สี, การจัดเรียง, ลิงก์, รายการ, รูปภาพจาก URL, วิดีโอ, คอมโพเนนต์, กริด และสลับ HTML

ไม่มีการส่งผ่านค่าเพิ่มเติม — แค่ value, placeholder, แหล่งรูปภาพ BsProse.imageUrl() และ onChange ทุก ตัวเลือกอื่น (grid, embed, sourceToggle) เปิดใช้งานโดยค่าเริ่มต้น 

// ตัวอย่างที่ 1 — การกำหนดค่าเริ่มต้น
BsProse.create('#editor-1', {
  value: '<h2>Hello!</h2>This is <b>BsProse</b>.',
  placeholder: 'เริ่มเขียนหน้าของคุณ…',
  image: { sources: [ BsProse.imageUrl() ] },
  onChange: html => out1.textContent = html
});

2. แถบเครื่องมือขั้นต่ำ (ความคิดเห็นบล็อก)

            

              toolbar แบบย่อ, ขนาด sm, ไม่มีคอมโพเนนต์, กริด,
              วิดีโอ หรือสลับ HTML
            

          

          

            

              เราส่งอาร์เรย์ toolbar ของเราเองและปิดใช้งานคุณสมบัติหนัก
              ปุ่มที่ไม่มีการกำหนดค่าจะถูกข้ามโดยอัตโนมัติ
            

            
// ตัวอย่างที่ 2 — ชุดเครื่องมือขนาดกะทัดรัด
BsProse.create('#editor-2', {
  toolbar: ['bold', 'italic', 'underline', '|',
            'link', 'bullet', 'number', '|', 'clean'],
  toolbarSize: 'sm',
  grid: false, embed: false, sourceToggle: false,
  placeholder: 'ความคิดเห็นของคุณ…',
  onChange: html => out2.textContent = html
});

3. หลายแหล่งที่มาของรูปภาพ

            

              URL, อัปโหลดไฟล์ (imageUpload) และแหล่ง "แกลเลอรี" กำหนดเอง
            

          

          

            

              เมื่อมีหลายแหล่ง ปุ่ม "รูปภาพ" จะแสดงตัวเลือกก่อน การอัปโหลดชี้ไปที่
              upload.php ที่มาพร้อมกัน แหล่งกำหนดเองเป็นเพียงออบเจ็กต์
              { id, label, open(api) } ที่เรียก api.insert(spec)
            

            
// ตัวอย่างที่ 3 — สามแหล่งรูปภาพ
const gallery = {
  id: 'gallery', label: 'คลังมีเดีย (สาธิต)',
  open(api) {
    api.insert({
      src: 'https://placehold.co/640x360',
      alt: 'รูปภาพสุ่ม', title: 'รูปภาพสุ่ม', class: 'img-fluid rounded shadow-sm'
    });
  }
};
BsProse.create('#editor-3', {
  image: { sources: [
    BsProse.imageUrl(),
    BsProse.imageUpload({ endpoint: 'upload.php' }),
    gallery
  ]},
  toolbar: ['format', '|', 'bold', 'italic', '|', 'align', 'link', '|', 'image'],
  onChange: html => out3.textContent = html
});

4. แค็ตตาล็อกคอมโพเนนต์กำหนดเอง

            

              ขยาย BsProse.components ด้วยเทมเพลต "การ์ดราคา" กำหนดเอง
            

          

          

            

              คอมโพเนนต์คือ { id, group, label, html } โทเค็น {{id}}
              ใดๆ ในเทมเพลตจะถูกแทนที่ด้วย id ที่ไม่ซ้ำกันเมื่อแทรก ที่นี่บล็อกการตลาด
              ถูกเพิ่มเข้าไปในชุดเริ่มต้น
            

            
// ตัวอย่างที่ 4 — ขยายชุดคอมโพเนนต์
BsProse.create('#editor-4', {
  components: [
    ...BsProse.components,                // เก็บค่าเริ่มต้นไว้
    {
      id: 'pricing', group: 'การตลาด', label: 'การ์ดราคา',
      html: '<div class="card text-center shadow-sm">' +
            '<div class="card-header">Pro</div>' +
            '<div class="card-body"><h3>$29</h3>' +
            '<a class="btn btn-primary">Choose</a></div></div>'
    }
  ],
  onChange: html => out4.textContent = html
});

6. รูปภาพรองรับหน้าจอ: srcset & <picture>

            

              สเปครูปภาพสามารถมี srcset/sizes (รูปภาพ <img>
              รับรู้ DPR) หรือ sources (รูปภาพ <picture> ขับเคลื่อนด้วย viewport)
            

          

          

            

              แหล่งรูปภาพใดๆ (URL, อัปโหลด หรือแกลเลอรีกำหนดเอง) แก้ไขเป็น
              สเปครูปภาพ นอกจาก src คุณสามารถเพิ่ม:
            

            
    
              
  • 
                srcset + sizes — โปรแกรมแก้ไขจะส่งออก
                <img srcset sizes> ธรรมดา เบราว์เซอร์เลือกตัวเลือกตาม
                ขนาดที่แสดง × device-pixel-ratio — เหมาะสำหรับ HiDPI/Retina
                แต่จะไม่ลดขนาดเมื่อไฟล์ใหญ่กว่าถูกแคชไว้แล้ว
              
    • 
              
  • 
                sources — อาร์เรย์ของ
                { media, type, srcset, sizes } โปรแกรมแก้ไขส่งออก
                <picture> พร้อมกฎ <source media>:
                การสลับ แบบเข้มงวดตาม viewport ที่ประเมินใหม่เมื่อปรับขนาด
                (หน้าต่างแคบลง → ไฟล์เล็กลง) <img> ธรรมดากลายเป็น
                ตัวเลือกสำรองที่ใหญ่ที่สุดและไม่มี srcset/sizes ของตัวเอง
              
    • 
            

            

              สองแหล่งกำหนดเองด้านล่างแทรกแต่ละตัวแปร เปิดตัวเลือกรูปภาพและเลือกหนึ่ง
              จากนั้นตรวจสอบผลลัพธ์ getHTML() เพื่อดู <img srcset>
              เทียบกับ <picture>
            

            
// ตัวอย่างที่ 6 — สเปครูปภาพรองรับหน้าจอสองแบบ

// (a) <img srcset sizes> รับรู้ DPR
const srcsetSource = {
  id: 'srcset', label: 'แทรก <img srcset>',
  open(api) {
    api.insert({
      src:    'https://placehold.co/1280x720',
      srcset: 'https://placehold.co/480x270 480w, ' +
              'https://placehold.co/960x540 960w, ' +
              'https://placehold.co/1280x720 1280w',
      sizes:  '(max-width: 960px) 100vw, 960px',
      width: 1280, height: 720, alt: 'รูปภาพรองรับหน้าจอ', title: 'รูปภาพรองรับหน้าจอ',
      class: 'img-fluid rounded shadow-sm'
    });
  }
};

// (b) <picture> ขับเคลื่อนด้วย viewport ผ่าน `sources`
const pictureSource = {
  id: 'picture', label: 'แทรก <picture>',
  open(api) {
    api.insert({
      src: 'https://placehold.co/1280x720',   // สำรอง / ใหญ่ที่สุด
      sources: [
        { media: '(max-width: 575.98px)',  srcset: 'https://placehold.co/480x270' },
        { media: '(max-width: 1199.98px)', srcset: 'https://placehold.co/960x540' }
      ],
      width: 1280, height: 720, alt: 'รูปภาพปรับตาม viewport', title: 'รูปภาพปรับตาม viewport',
      class: 'img-fluid rounded shadow-sm'
    });
  }
};

BsProse.create('#editor-6', {
  image: { sources: [ srcsetSource, pictureSource ] },
  toolbar: ['format', '|', 'bold', 'italic', '|', 'align', '|', 'image'],
  onChange: html => out6.textContent = html
});

            
            

            

              ผลลัพธ์ getHTML()
              

            

            

              หลักการ: ส่งเฉพาะ srcset สำหรับค่าเริ่มต้นรับรู้ DPR; ส่ง
              sources เมื่อต้องการสลับแบบเข้มงวดและปรับแต่งศิลปะตาม viewport
            

          

        

      


      
      

        
สรุปตัวเลือก

        

          

            
            
ตัวเลือกประเภทค่าเริ่มต้นความหมาย

            
            
            
valuestring''HTML เริ่มต้น

            
onChange(html)=>void—เรียกเมื่อมีการเปลี่ยนแปลง (debounce ~120ms)

            
placeholderstringStart writing…แสดงเมื่อโปรแกรมแก้ไขว่างเปล่า

            
toolbarstring[]ชุดเต็มลำดับโทเค็นแถบเครื่องมือ; '|' คือตัวคั่น

            
componentsComponent[]BsProse.componentsชุดคอมโพเนนต์; [] ซ่อนปุ่ม

            
image{ sources:[] }undefinedแหล่งรูปภาพ; หากไม่มีจะซ่อนปุ่ม

            
gridbooleantrueตัวสร้างกริดแบบภาพ

            
embedbooleantrueแทรกวิดีโอในกล่อง ratio

            
toolbarSizesm|md|lgmdขนาดตัวควบคุมแถบเครื่องมือ

            
historyLimitnumber120ความลึกของ undo/redo

            
sourceTogglebooleantrueสลับ HTML-source (ปักหมุดทางขวาเสมอ)

            
          

        

        

          โทเค็นแถบเครื่องมือ: format, bold, italic,
          underline, color, bgcolor, align,
          link, bullet, number, image,
          embed, component, grid, clean,
          |
        

        

          ฟิลด์สเปครูปภาพ: src (จำเป็น), srcset, sizes
          (→ <img> รองรับหน้าจอ), sources (→ <picture>),
          alt, title, width, height, class ดูตัวอย่างที่ 6
